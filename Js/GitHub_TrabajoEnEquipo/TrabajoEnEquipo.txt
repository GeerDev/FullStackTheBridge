Trabajo en equipo utilizando GitHub:

    - PR -> Pull Request -> Una propuesta para implementar código en un proyecto
    - Los colaboradores los puedes agregar tú a tus propios repositorios -> Ojo que pueden aceptar o denegar PR's (incluso las suyas)
    - Las PR's se tienen que aceptar o denegar
    - El repositorio al detectar "fork" o "presencia de colaboraciones" al hacer "push" directamente te da la opcion de hacer una PR en la mayoria de casos
    - Es buena práctica crear una rama aparte para hacer los cambios y subir la rama completa
